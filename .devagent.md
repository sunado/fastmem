# FastMem Development Agent

This agent manages the development environment for the FastMem flashcard application.

## Capabilities

### Server Management
- **Start dev server**: `dev:start` - Launches Vite dev server on http://localhost:5173
- **Stop dev server**: `dev:stop` - Gracefully stops the running dev server
- **Dev server logs**: `dev:logs` - Shows current dev server output
- **Restart dev server**: `dev:restart` - Stops and restarts dev server

### Database Operations
- **Initialize database**: `db:init` - Creates SQLite database and seeds default user (user/user)
- **Migrate database**: `db:migrate` - Runs database migrations
- **Inspect database**: `db:inspect` - Opens SQLite CLI for direct database inspection

### Build & Quality
- **Build production**: `build:prod` - Creates optimized production build
- **Preview production**: `build:preview` - Serves production build locally
- **Type check**: `check:types` - Runs TypeScript and Svelte type checking
- **Type check watch**: `check:watch` - Continuous type checking with file watching
- **Lint code**: `lint` - Runs ESLint on codebase
- **Format code**: `format` - Auto-formats code with Prettier

### Project Info
- **Dev status**: `status` - Shows current dev environment status and port
- **Show scripts**: `show:scripts` - Lists all available npm scripts
- **Show ports**: `show:ports` - Lists services running on standard ports

## Default Credentials

- **Username**: `user`
- **Password**: `user`

## Project Structure

- **Frontend**: SvelteKit components in `src/lib/components/`
- **API Routes**: Endpoints in `src/routes/api/`
- **Database**: SQLite at `data/fastmem.db` with Drizzle ORM
- **Stores**: State management in `src/lib/stores/`
- **Utils**: Helpers in `src/lib/utils/`

## Common Workflows

### First-Time Setup
```
1. db:init      # Initialize database with seed data
2. dev:start    # Start development server
3. Open http://localhost:5173 in browser
4. Login with user/user
```

### Development Loop
```
1. Make code changes in src/
2. Server hot-reloads automatically (dev:start required)
3. Check types: check:types
4. Lint: lint
5. Format: format
6. Test changes in browser
```

### Production Deployment
```
1. format       # Format code
2. lint         # Check quality
3. check:types  # Verify types
4. build:prod   # Create production build
5. build:preview # Test production build locally
```

## Environment Variables

Create `.env.local` in project root if needed:

```
# Database (SQLite path)
DATABASE_URL=data/fastmem.db

# Server
VITE_API_BASE=http://localhost:5173
```

## Database Tables

- **users**: User accounts with hashed passwords
- **flashcard_sets**: Collections of flashcards
- **flashcards**: Individual cards with Q&A pairs

Inspect with: `sqlite3 data/fastmem.db`

## Port Configuration

- **Dev Server**: http://localhost:5173
- **API**: Same as dev server (http://localhost:5173/api/*)

## Troubleshooting

- **Port already in use**: `dev:stop` then `dev:start`
- **Database locked**: Ensure only one process accesses `data/fastmem.db`
- **Type errors**: Run `check:types` to see detailed TypeScript errors
- **HMR issues**: Restart dev server with `dev:restart`
- **Stale builds**: Clear `dist/` and `.svelte-kit/` folders before `build:prod`
